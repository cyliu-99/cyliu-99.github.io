Vim Markdown 方案与v2ray订阅解析

Posté le 2021-04-04

摸鱼的一天，没怎么看论文，在折腾一些软件。

vim markdown-preview插件

将原来的jupyter lab换成了vim的markdown-preview插件，虽然jupyter lab很好用，配上vim键位，也可以实现很多vim快捷键，但是每次更新还要重新build，感觉不是很友好，另外，文件太长了渲染很慢，很卡。markdown-preview插件响应迅速多了，此外界面很简洁，数学公式默认katex渲染，速度更快，最重要的，预览会跟随vim光标移动。 内存占用方面，vim几乎不占内存，markdown-preview后台是node，占用内存大概700M。由于装完markdown-preview插件后就把jupyter全删了，没来得及比较jupyter内存占用。Anyway，反正电脑内存够用，内存占用多一点也无所谓。

然而markdown-preview插件无法在安卓的termux中使用，github上有人提类似issue，应该是node的问题，不过termux连上蓝牙键盘后会自动进入latin mode，无法输入中文，就算能安装了打不了中文也很难受，所以安卓还是jupyter lab方案。最近华为系统更新，杀后台更加严重，jupyter有时候写的内容没保持上，也是一个痛点，想找个好用编辑器太难了。
